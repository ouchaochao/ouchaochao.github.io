自动部署hugo脚本

Written by Chaochao Ou with ♥ on August 18, 2019 in

写完markdowm再输入手动命令行进行部署hugo太费事了，于是就写了这个脚本：

#!/bin/sh
input = ""

function list(){
echo "查询到有以下md文件:"
cd content/post/
ls
cd ../../
}

function add(){
echo "请输入文章的题目："
read input
hugo new post/$input.md
vim content/post/$input.md
hugo
cd public
git add .
git commit -m "add $input"
git push origin master
echo ""
echo "Add $input done!"
}

function deploy(){
hugo
cd public
git add .
git commit -m "add $input"
git push origin master
echo ""
echo "Deploy done!"
}

function change(){
list
echo "请输入要修改的文章题目："
read input    
vim content/post/$input.md
hugo
cd public
git add .
git commit -m "change $input"
git push origin master
echo ""
echo "Change $input done!"

}

function del(){
list
echo "请输入要删除的文章题目："
read input    
rm content/post/$input.md
hugo
cd public
git add .
git commit -m "del $input"
git push origin master
echo ""
echo "Del $input done!"
}

echo "请输入序号："
echo "    1.新建post"
echo "    2.修改post"
echo "    3.删除post"
echo "    4.部署"
echo "    5.退出"
read b

if (( $b==1 ));then
add
elif (( $b==2 ));then
change
elif (( $b==3 ));then
del
elif (( $b==4 ));then
deploy
else
echo "退出成功"
fi

目录结构如下：

myblog
├── archetypes
├── config.toml
├── content
├── data
├── layouts
├── new.sh
├── public
├── resources
├── static
└── themes

我把它放在了myblog文件夹下，命名为new.sh，当我要新建/修改/删除博客的时候就只要输入./new.sh就行了，下面是操作示例：

新建

myblog ./new.sh
